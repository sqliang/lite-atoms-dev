# Code Review Best Practices

## PR Requirements

- [ ] Clear title describing the change
- [ ] Detailed description explaining context and impact
- [ ] Single logical change per PR
- [ ] All CI checks passed
- [ ] At least one reviewer approval

## Review Process

- Review promptly once assigned
- Focus on: code correctness, project conventions, performance, test coverage, security
- Be specific and constructive in feedback
- Approve only when all concerns are resolved

Reference: [Google's Code Review Guide](https://google.github.io/eng-practices/review/reviewer/)
